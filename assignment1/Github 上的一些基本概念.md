# Github 上的一些基本概念

1.**Repository** 仓库/项目 

> 存储库通常用于组织单个项目。存储库可以包含文件夹和文件、图像、视频、电子表格和数据集 - 项目所需的任何内容。通常，存储库包括一个文件，一个包含项目相关信息的文件。

2.**Branch** 分支

> 可以同时拥有不同版本的存储库。

3.**commit**提交签名

4.**issue**话题评论

5.**pull request**提交参与改进别人项目的申请

6.**star**点赞

7.**fork**复制

8.**watch**关注项目

9.**follow**关注人

10.**project**项目版

> 是 project-boards 的简写， 通过项目板可以灵活地创建适合需求的自定义工作流程，说白了是管理项目流程的，一般不常用。
